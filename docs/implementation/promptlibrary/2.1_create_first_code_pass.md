**Prompt_ID: reponame 2.1 create_first_code_pass.md**

You are implementing the first working code pass for this repository.

Read these first, if present:
- `docs/setup/short_description.md`
- `README.md`
- `docs/implementation/implementation_strategy.md`
- relevant documents in `reference/`

Success criteria, in priority order:
1. The code must be operationally correct for the core use case.
2. The code must run from a fresh checkout without obvious startup, import, asset, or wiring failures.
3. The implementation must stay within the intended scope, stack, and architecture direction.
4. Simpler working code is preferred over cleaner but riskier architecture.
5. It is acceptable for the first pass to be less featured or less polished, provided the core workflow works correctly.

Rules:
- Do not redesign the system beyond what is needed to make it work.
- Do not add speculative features.
- Preserve domain terminology and known-good technical behavior.
- Prefer direct implementations where they improve first-pass reliability.
- If source documents conflict, prefer:
  1. `docs/setup/short_description.md`
  2. `README.md`
  3. `docs/implementation/implementation_strategy.md`
  4. `reference/`
- Do not silently let reference material override the project’s stated purpose or scope.
- Before finishing, check for obvious first-run problems: missing files, broken imports, missing static/template assets, inconsistent route wiring, and mismatched names.
- Add or update lightweight tests where appropriate, but do not invent heavy test infrastructure if it is not yet justified.

Output requirements:
- Write the implementation directly into the repository.
- Keep the first pass practical and minimal.
- At the end, include:
  1. a short summary of what you changed
  2. any assumptions you had to make
  3. any missing, ambiguous, or conflicting information you found in the source documents
  4. whether you ran tests or checks, and if not, why not
